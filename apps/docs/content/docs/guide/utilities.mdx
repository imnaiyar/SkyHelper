## Timestamp Utility

Convert dates and times to Discord-formatted timestamps for perfect scheduling.

### Command

```
/utils timestamp time:HH mm ss
```

**Options**:

| Option       | Description                                                                                   |
| ------------ | --------------------------------------------------------------------------------------------- |
| `<time>`     | The time to convert to UNIX (format: `HH mm ss`, e.g., `04 32 44`)                            |
| `[timezone]` | The timezone to use for conversion (format: `Continent/City`, default: `America/Los_Angeles`) |
| `[date]`     | The date to convert (format: `DD`)                                                            |
| `[month]`    | The month to convert (format: `MM`)                                                           |
| `[year]`     | The year to convert (format: `YYYY`)                                                          |

**[Image Placeholder: Timestamp converter]**

### Usage Examples

<Steps>

<Step>

#### Basic Time

```
/utils timestamp time:02 07 44
```

Converts 02:07:44 in default timezone (America/Los_Angeles).

</Step>

<Step>

#### With Timezone

```
/utils timestamp time:04 22 30 timezone:Asia/Kolkata
```

Converts 04:22:30 in Asia/Kolkata timezone.

</Step>

<Step>

#### Full Date and Time

```
/utils timestamp time:15 30 00 date:25 month:12 year:2024
```

Converts December 25, 2024 at 15:30:00.

</Step>

</Steps>

### Why This Format?

<Callout type="info">
  **Space Separator**: The format `HH mm ss` (with spaces) is used instead of `HH:mm:ss` because Discord formats `:xx:` into
  custom emoji format if it matches any emojis accessible to the user. This is a workaround to prevent that issue.
</Callout>

### Discord Timestamp Formats

The command provides all Discord timestamp formats:

- **Short Time**: `16:20`
- **Long Time**: `16:20:30`
- **Short Date**: `20/04/2021`
- **Long Date**: `20 April 2021`
- **Short Date/Time**: `20 April 2021 16:20`
- **Long Date/Time**: `Tuesday, 20 April 2021 16:20`
- **Relative Time**: `2 months ago`

### Timezone Support

Supported timezone format: `Continent/City`

**Examples**:

- `America/Los_Angeles`
- `America/New_York`
- `Europe/London`
- `Europe/Paris`
- `Asia/Tokyo`
- `Asia/Kolkata`
- `Australia/Sydney`

<Callout type="tip">
  **Find Your Timezone**: Use the format `Continent/City`. For example, if you're in Mumbai, use `Asia/Kolkata`.
</Callout>
