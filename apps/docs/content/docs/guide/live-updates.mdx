# Live Updates

Set up auto-updating messages that display real-time shard information and event schedules in your Discord server.

## Overview

Live updates provide persistent, automatically refreshing messages that keep your server informed about:

- Current shard information
- Event schedules and countdowns
- Real-time updates without command spam

<Callout type="error">
  **Server Only**: Live updates are **only available for servers** (not user installations) and require **Manage Server**
  permissions to configure.
</Callout>

<Callout type="info">
  **Webhook Required**: The bot needs **Manage Webhooks** permission in the channel where live updates will be posted.
</Callout>

## Commands

### `/shards-live` - Live Shard Updates

Set up auto-updating shard information messages.

#### Start Live Updates

**Options**:

| Option      | Description                                              |
| ----------- | -------------------------------------------------------- |
| `<channel>` | The channel where the live update message should be sent |

**Usage Example**:

```
/shards-live start channel:#live-shards
```

Starts live shard updates in the #live-shards channel.

## How Live Updates Work

### Setup Process

<Steps>

<Step>

#### Run Start Command

Execute the start command with your chosen channel.

</Step>

<Step>

#### Webhook Creation

The bot creates a webhook in the specified channel named "SkyHelper Live Updates".

</Step>

<Step>

#### Initial Message

An initial message is posted with current information.

</Step>

<Step>

#### Automatic Updates

The message updates automatically at regular intervals.

</Step>

</Steps>

### Update Frequency

- **Shards Live**: Updates every **5 minutes**
- **Skytimes Live**: Updates every **2 minutes**

<Callout type="info">
  Update intervals are optimized to balance freshness with Discord API rate limits. More frequent updates could cause issues.
</Callout>

### Channel Permissions

Recommended permissions:

- **Members**: Can view, cannot send messages
- **Bot**: Can view, send messages, manage webhooks
- **Admins**: All permissions

This prevents chat clutter in live update channels.

<Callout type="tip">
  **Read-Only Channels**: Make live update channels read-only for members to avoid message clutter between updates.
</Callout>

## Managing Live Updates

### Viewing Active Updates

Check if live updates are active:

1. Look for the webhook in channel settings
2. Check for the live message in the channel
3. Verify it's updating (check timestamp)

### Updating Configuration

To change the channel:

<Steps>

<Step>

Stop current live updates:

```
/shards-live stop
```

</Step>

<Step>

Start in new channel:

```
/shards-live start channel:#new-channel
```

</Step>

</Steps>

### Multiple Live Updates

You can have:

- One shards live update per server
- One skytimes live update per server
- Both types in the same or different channels

<Callout type="warning">
  **One Per Type**: You can only have one active shards-live and one active skytimes-live per server. Starting a new one
  automatically stops the previous.
</Callout>

## Webhook Details

### Webhook Creation

When you start live updates:

<Steps>

<Step>

Bot checks for existing "SkyHelper Live Updates" webhook

</Step>

<Step>

If not found, creates a new webhook

</Step>

<Step>

Webhook is used to post the live message

</Step>

<Step>

Webhook ID is stored for future updates

</Step>

</Steps>

### Webhook Management

<Callout type="warning">
  **Do Not Delete Webhook**: If you delete the "SkyHelper Live Updates" webhook manually, live updates will automatically stop.
</Callout>

If webhook is deleted:

1. Live updates stop automatically
2. Run the start command again
3. New webhook is created
4. Live updates resume

## Update Reliability

### When Updates Happen

Updates occur:

- Every 5 minutes (shards)
- Every 2 minutes (skytimes)
- Automatically in the background
- Even if bot appears offline temporarily

### Update Failures

If an update fails:

- Bot retries automatically
- Next scheduled update continues
- Error logged for debugging
- No notification to users

<Callout type="info">
  **Temporary Failures**: Occasional update failures are normal and self-recover. If updates stop completely, restart the live
  update.
</Callout>

## Troubleshooting

### Live Updates Not Updating

<Steps>

<Step>

#### Check Timestamp

Look at "Last Updated" in the message. If it's very old, updates may have stopped.

</Step>

<Step>

#### Verify Webhook

Check channel settings for "SkyHelper Live Updates" webhook.

</Step>

<Step>

#### Check Permissions

Ensure bot has Manage Webhooks permission.

</Step>

<Step>

#### Restart Updates

Stop and start the live update to reinitialize.

</Step>

</Steps>

### Message Deleted

<Callout type="warning">
  **Auto-Disable**: If the live message is deleted, live updates automatically stop. Run the start command again to create a new
  message.
</Callout>

### Wrong Channel

To move live updates:

```
/shards-live stop
/shards-live start channel:#correct-channel
```

### Permission Errors

If you get permission errors:

1. Verify Manage Server permission
2. Check bot's Manage Webhooks permission in channel
3. Ensure channel isn't in a restricted category
